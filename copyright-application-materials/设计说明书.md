## 一、项目概述

### 1.1 项目背景

随着现代社会生活节奏加快，人们面临的心理压力越来越大，对心理健康的关注度不断提升。传统的心理咨询服务存在成本高、预约难、隐私顾虑等问题。同时，虚拟现实技术和人工智能技术的快速发展为创新型心理健康应用提供了技术基础。

心灵絮语项目应运而生，旨在利用WebVR技术和大语言模型AI技术，打造一个低门槛、高沉浸感的AI心理陪伴平台，让用户在轻松愉悦的3D虚拟环境中获得情感支持。

### 1.2 设计目标

1. **技术目标**
   - 实现流畅的WebVR 3D场景渲染
   - 实现实时流式AI对话
   - 实现用户个性化配置系统
   - 保证系统稳定性和可扩展性

2. **用户体验目标**
   - 提供沉浸式的虚拟环境体验
   - 提供自然流畅的AI对话体验
   - 提供简单易用的操作界面
   - 保护用户隐私和数据安全

3. **功能目标**
   - 支持用户注册和登录
   - 支持AI智能对话
   - 支持世界自定义配置
   - 支持文件上传管理
   - 支持世界分享功能

### 1.3 技术选型

**前端技术栈:**
- A-Frame 1.4+: WebVR框架，用于3D场景构建
- 原生JavaScript: 前端逻辑实现
- HTML5 + CSS3: 页面结构和样式
- WebSocket: 实时通信协议

**后端技术栈:**
- Python 3.8+: 后端开发语言
- FastAPI: 现代异步Web框架
- SQLModel: ORM框架
- SQLite/MySQL: 数据库
- Anthropic Claude API: AI对话模型
- PyYAML: 配置文件解析

**部署与运维:**
- Uvicorn: ASGI服务器
- Nginx: 反向代理和静态文件服务
- SSL/TLS: HTTPS加密通信

## 二、系统架构设计

### 2.1 总体架构

系统采用前后端分离的B/S架构，前端通过HTTP/WebSocket与后端API通信。

```
┌─────────────────────────────────────────────────────────┐

│  └──────────────┘  └──────────────┘  └──────────────┘  │
└──────────────┬──────────────┬──────────────┬───────────┘
               │              │              │
         HTTP/HTTPS      RESTful API    WebSocket
               │              │              │
┌──────────────┴──────────────┴──────────────┴───────────┐
│                    FastAPI后端服务                       │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │  路由控制器  │  │  业务逻辑层  │  │  WebSocket   │  │
│  │  (Routes)    │  │  (Services)  │  │   处理器     │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
└──────┬───────────────────┬───────────────────┬─────────┘
```

## 三、数据库设计

### 3.1 数据库选型

系统支持SQLite和MySQL两种数据库：
- **开发环境**: SQLite（轻量级、免配置）
- **生产环境**: MySQL（高性能、支持并发）

### 3.2 数据表设计

#### 3.2.1 用户表 (user)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY | 用户ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引:**
- PRIMARY KEY (id)
- UNIQUE INDEX (username)

#### 3.2.2 世界配置表 (world)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY | 世界ID |
| user_id | INTEGER | FOREIGN KEY | 所属用户ID |
| name | VARCHAR(100) | DEFAULT '我的世界' | 世界名称 |
| sky_texture | VARCHAR(255) | NULL | 天空贴图路径 |
| fairy_model | VARCHAR(255) | NULL | 精灵模型路径 |
| background_music | VARCHAR(255) | NULL | 背景音乐路径 |
| system_prompt | TEXT | NULL | 系统提示词 |
| greeting_message | TEXT | NULL | 初次见面语 |
| is_public | BOOLEAN | DEFAULT FALSE | 是否公开 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引:**
- PRIMARY KEY (id)
- INDEX (user_id)
- FOREIGN KEY (user_id) REFERENCES user(id)

#### 3.2.3 聊天记录表 (chat)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY | 记录ID |
| user_id | INTEGER | FOREIGN KEY | 用户ID |
| world_id | INTEGER | FOREIGN KEY, NULL | 世界ID |
| role | VARCHAR(20) | NOT NULL | 角色(user/assistant) |
| content | TEXT | NOT NULL | 消息内容 |
| timestamp | DATETIME | DEFAULT CURRENT_TIMESTAMP | 时间戳 |

**索引:**
- PRIMARY KEY (id)
- INDEX (user_id)
- INDEX (world_id)
- INDEX (timestamp)

#### 3.2.4 上传记录表 (uploadrecord)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY | 记录ID |
| user_id | INTEGER | FOREIGN KEY | 用户ID |
| world_id | INTEGER | FOREIGN KEY, NULL | 世界ID |
| file_type | VARCHAR(50) | NOT NULL | 文件类型 |
| original_filename | VARCHAR(255) | NOT NULL | 原始文件名 |
| stored_filename | VARCHAR(255) | NOT NULL | 存储文件名 |
| file_path | VARCHAR(255) | NOT NULL | 文件路径 |
| file_size | INTEGER | NOT NULL | 文件大小(字节) |
| uploaded_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 上传时间 |

**索引:**
- PRIMARY KEY (id)
- INDEX (user_id)
- INDEX (world_id)

#### 3.2.5 反馈表 (feedback)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY | 反馈ID |
| user_id | INTEGER | FOREIGN KEY | 用户ID |
| feedback_type | VARCHAR(50) | NOT NULL | 反馈类型 |
| title | VARCHAR(200) | NOT NULL | 反馈标题 |
| content | TEXT | NOT NULL | 反馈内容 |
| status | VARCHAR(20) | DEFAULT 'pending' | 状态 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**索引:**
- PRIMARY KEY (id)
- INDEX (user_id)
- INDEX (status)

#### 3.2.6 世界分享表 (worldshare)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY | 分享ID |
| world_id | INTEGER | FOREIGN KEY | 世界ID |
| share_code | VARCHAR(20) | UNIQUE, NOT NULL | 分享码 |
| view_count | INTEGER | DEFAULT 0 | 浏览次数 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| expires_at | DATETIME | NULL | 过期时间 |

**索引:**
- PRIMARY KEY (id)
- UNIQUE INDEX (share_code)
- INDEX (world_id)

## 四、功能模块详细设计

### 4.1 用户管理模块

#### 4.1.1 用户注册/登录流程

```
用户输入用户名
    │
    ▼
POST /api/users/register
    │
    ├─→ 用户名已存在 ──→ 返回用户ID (is_new=false)
    │
    └─→ 用户名不存在 ──→ 创建新用户
                          │
                          ├─→ 创建默认世界
                          │
                          └─→ 返回用户ID (is_new=true)
```

#### 4.1.2 关键代码逻辑

```python

    # 1. 查询用户是否存在
    # 2. 如果存在，直接返回
    # 3. 如果不存在，创建用户和默认世界
    # 4. 返回用户信息
```

## 六、安全性设计

### 6.1 数据安全

1. **传输安全**
   - 生产环境强制使用HTTPS
   - WebSocket使用WSS加密传输
   - API密钥不在前端暴露

2. **存储安全**
   - 数据库密码环境变量配置
   - 文件上传路径限制在uploads目录
   - 用户数据隔离存储

3. **隐私保护**
   - 对话记录只有用户本人可见
   - 世界配置默认私密
   - 不收集用户敏感信息

### 6.2 输入验证

1. **文件上传验证**
   - 文件类型白名单验证
   - 文件大小限制（50MB）
   - 文件扩展名检查
   - MIME类型验证

2. **参数验证**
   - 使用Pydantic进行请求参数验证
   - SQL注入防护（使用ORM）
   - XSS防护（前端输入过滤）

### 6.3 访问控制

1. **用户鉴权**
   - 用户名作为身份标识
   - 每个请求验证用户权限
   - 只能访问自己的资源

2. **资源隔离**
   - 用户只能修改自己的世界
   - 文件按用户ID隔离存储
   - 访客模式功能受限

### 6.4 错误处理

1. **异常捕获**
   - 全局异常处理器
   - WebSocket异常处理
   - 文件操作异常处理

2. **错误日志**
   - 记录关键操作日志
   - 记录错误堆栈信息
   - 方便问题排查
